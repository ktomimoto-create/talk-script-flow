    - 「登録方法等」を「登録方法」に名称変更しました。
    - オンライン（カスタマーサイト）とオフライン（ロッカー操作）それぞれの案内手順を追加しました。
- **ヒアリング項目の視認性微調整**
    - 矢印（→）を伴うヒアリング項目の見出しテキストが太すぎたため、他の項目に合わせて太さを調整（700→500）しました。
- **「不具合の通報」ヒアリング内容の更新**
    - 「不具合の通報」項目において、内容の聞き取りから、号機・物件名の確認およびコントロールセンターへの連絡先案内に変更しました。（管理会社・管理員の双方に適用）
- **「報告書に関する問い合わせ」および「メンテナンス日の確認」ヒアリング内容の更新**
    - 「メンテナンス日の確認」項目において、「過去の保守点検日」を「過去の定期点検日」に、「保守点検の予定日」を「定期点検予定日」に変更し、表記を統一しました。（管理会社・管理員の双方に適用）
    - 「定期メンテナンス報告書が送られてきていない」旨の問い合わせに対し、複数の回答パターン（アカウント申請の推進、データ送付要望時の引き継ぎ）をわかりやすく分割表示するようにしました。
    - 特記事項や不具合の確認、メンテナンス作業の対応時間確認の文章を整理し、視認性を向上させました。
    - ヒアリング項目の複数行（改行）や複数矢印（→）の表示に対応できるよう、プログラムのレンダリング処理を改良しました。

    - 最終ヒアリング画面最下部の「●●へ転送」ボタンを全て削除しました。
    - 各詳細項目の案内文を「以下の内容を確認のうえ、●●へ転送してください」から「以下の内容を確認のうえ、●●へ引き継ぎしてください」に統一しました。

- **架電メモ機能の拡張**
    - 架電メモの入力欄に「架電者」を追加し、誰が架電して不通になったかを記録できるようにしました。

## [2026-04-23]
- **「架電メモ」機能の搭載とクイックアクションの刷新**
    - 画面下部に常時表示されていたクイックアクション（ディスパ、保守管理、施工管理、他部署）を削除しました。
    - 代わりに、電話番号と名前を入力・保存できる「架電メモ」バーを新たに搭載しました。
    - **履歴一覧表示機能**: 保存した架電メモを一覧で確認できる機能を追加しました。不要な履歴の削除も可能です。
    - **不通時の履歴保存**: 相手が不在等で繋がらなかった際、「不通」ボタンを押すことで、その架電情報をアプリ内に一時保存できるようにしました。（文言を「繋がらなかった（保存）」から変更）
    - **折り返し電話の自動照合機能**: 受電時のメモ欄で「折り返し先」に電話番号を入力した際、保存済みの架電メモと一致する場合、誰がその番号に架電したかを「架電者: [名前]」として即座に表示する機能を実装しました。
    - **データの永続化**: 保存した架電メモは `localStorage` に記録されるため、ブラウザを閉じたり再読み込みしたりしても保持されます。

    - 管理会社、管理員、協力会社、建築、連動相手のヒアリングページにおいて、オペレーターが情報を一時的に記録できるフォームを表示。

        - 建築 / 連動相手: 「現場名」「会社名/氏名」「内容」「折り返し先」

- **パフォーマンス（動作）の軽量化最適化**
    - 全体フロー図の各ボックスに適用されていた重い描画処理（`backdrop-filter: blur`）を整理し、GPU負荷を大幅に軽減しました。
    - CSSの遷移アニメーション（`transition`）を、必要なプロパティのみに限定することで描画計算を効率化しました。
    - Reactコンポーネントのメモ化（`memo`）とイベントハンドラの最適化（`useCallback`）を行い、不要な再描画を抑制しました。
    - 主要なアニメーション要素に `will-change` を指定し、ブラウザによる描画最適化を促すようにしました。
- **フロー図（VisualFlow）の接続線ずれ修正と動的化**
    - 固定配置されていた接続線を廃止し、CSSの疑似要素（`::before`, `::after`）を用いて、各ボックスの中心から動的に線を生成する方式に変更しました。
    - これにより、項目の数やブラウザの幅に関わらず、常にボックスの中央に線が正確に接続されるようになりました。
    - 第2層から第3層（子項目）への接続線が不足していた問題を解消し、視覚的なフローの繋がりを改善しました。
    - 階層間の高さ調整を行い、受電スタートから末端までの線が不自然に重なったり空白ができたりしないよう最適化しました。
- **フロー図（VisualFlow）の表示スタイル調整**
    - 「協力会社」カテゴリ配下の「AFC、BLP等」や「FRESH ROOM、設置業者等」を含む、第3階層の全ラベルの文字色を「白（#ffffff）」に固定し、背景色に関わらず視認性を確保しました。
    - 「ディスパッチャー」や「施工管理」といった中継カテゴリーを囲んでいた枠線（ボーダー）を削除し、より洗練されたフラットなデザインに変更しました。
