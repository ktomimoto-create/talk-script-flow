# 仕様書：トークスクリプトフロー

## プロジェクト概要
株式会社フルタイムシステムの電話対応を補助するための、インタラクティブなトークスクリプトフローアプリケーション。

## 技術スタック
- **Frontend**: React (スタンドアロン版は CDN 経由、開発版は Vite)
- **Language**: TypeScript / JavaScript (Babel)
- **Styling**: Vanilla CSS (Glassmorphism デザイン)
- **Database / Backend**: Firebase (Cloud Firestore)
- **SSO Authentication**: Microsoft Entra ID (Azure AD)
- **Development Port**: 5177 (Vite 開発サーバー)

## アプリケーション構成

1.  **受電 (Main Entry)**: 最初のエントリーポイント。「受電」とその下の8カテゴリー。
2.  **フロー図 (地図)**: 各カテゴリーごとの主な転送先・用件を表示。管理会社、管理員、協力会社、建築、連動相手など、特定カテゴリに応じた分岐を視覚化。
3.  **詳細ヒアリング階層**: 各項目をクリックした際に表示される具体的な確認事項。
4.  **最終アクション**: 不要になったため削除（元の転送完了画面）。
5.  **メモ入力スペース**: 管理会社、管理員、協力会社、建築、連動相手のヒアリングページにおいて、オペレーターが情報を一時的に記録できるフォームを表示。
    - **カテゴリによる項目名の自動切り替え**:
        - 建築 / 連動相手: 「現場名」「会社名/氏名」「内容」「折り返し先」
        - 協力会社（FRESH ROOM、設置業者等）: 「号機/現場名」「会社名/氏名」「内容」「折り返し先」
        - 管理会社 / 管理員 / 協力会社（AFC、BLP等）: 「号機/物件名」「会社名/氏名」「内容」「折り返し先」
    - **「架電の折り返し」ボックスの仕様**:
        - 各カテゴリの引き継ぎ先（ディスパッチャー、施工管理）に配置される「架電の折り返し」ボックス。
        - 確認ポイントは「誰から連絡があったか\n→個人名が分かればそちらへ引き継ぎ」（改行と引き継ぎアクションの付与）とし、「何の案件に関する折り返しか」（カッコ書きを削除）に変更。

### 特徴
- **インタラクティブ・マップ**: 全体像を構造的に表示し、オペレーターが視覚的に判断・操作できるインターフェース。
- **下部メモバー（架電メモ＆個人メモの一体化表示）**: 画面下部にスッキリした1行のコンパクトなバーを常時表示。
    - **中央 (架電メモ)**: 電話番号、架電先氏名、号機/物件名を入力して「不通」履歴として保存できる機能。入力エリア全体を画面の中央にセンタリング配置し、過去の不通履歴一覧（パネル）もその直上の画面中央にポップアップ表示されるように設計。
    - **右側 (個人メモ表示)**: 「個人メモ履歴」ボタンを右端に配置。
        - ボタンを押すと、下部バーの上に「個人メモ履歴一覧パネル」がふわっと浮き出て表示される（不通履歴と同様のポップアップパネル）。
        - パネル内には、ログインユーザーが登録した個人メモが簡易リスト（1行）形式で表示される（初期画面では全件、用件ボックス選択中は該当ボックスに自動で絞り込み）。
        - パネル内の各リスト項目をクリックすると、画面中央にグラスモーフィズムデザインの「個人メモ詳細」ポップアップモーダルが開き、詳細情報（現場名、会社名、用件内容、折り返し番号、保存日時）の確認、およびその場での削除・閉じる操作が可能。
    - **自動照合**: 受電メモの「折り返し先」入力時に保存済み架電データと自動照合して架電者をバッジで表示。
- **ブラウザ履歴連携**: ブラウザの「戻る」「進む」ボタンに対応。各状態がURLハッシュ（#id）として保存。詳細ウィンドウが開いている状態での別の用件のクリックは履歴を上書き（replaceState）し、サイドバー内の「戻る」ボタンを押した際は一発で初期画面（受電スタート）に戻ってウィンドウを閉じる設計に最適化。
- **動的カラーリング**: 選択したカテゴリーに応じたテーマカラーが適用。
- **洗練されたグループ表示**: 中間カテゴリー（ディスパッチャー、施工管理等）は、背景の囲みや枠線を廃止し、テキストのみのフラットなデザインで表示。
- **動的コネクタ**: ボックスの数や配置に合わせ、接続線を動的に生成。
- **SSO 認証 (Microsoft Entra ID)**: Microsoft アカウントによるサインイン機能を搭載。
    - **AuthGate**: 未ログイン時はサインイン画面を表示。
    - **プロフィールの表示**: 右上チップにユーザー名とメールアドレスを表示。
- **データ保護と自動削除**:
    - **プライバシー**: 架電メモはサインインしたユーザー本人のもののみが表示される。
    - **24時間自動削除**: 保存から24時間が経過したメモは一覧から自動的に除外される。
- **操作性向上 (Hover Effect)**: フロー図内の各サブ項目（詳細な用件）にカーソルを合わせた際、背景の強調、拡大、シャドウ付与が行われ、選択中であることが明確に分かるように改善。
- **画面サイズの固定化**: スケールを「中（PC）」の比率に固定し、常に安定したレイアウトで表示されるように設計。
- **テーマ切り替え機能（ライトモード・ダークモード）**:
    - 画面最下部の「架電メモバー（下部バー）」の右側（「集計ダッシュボード」ボタンの左隣）にテーマ切り替えボタン（太陽/月アイコン）を配置。
    - 選択したテーマは `localStorage` に保存され、ページのリロード後も設定が維持される。
    - ダークモード（デフォルト：宇宙的な深みのある紺色のグラデーション）に加え、プレミアムな質感（ガラスモーフィズム）を維持したライトモード（Apple調のすっきりした白〜淡いグレーのグラデーション背景とダークグレーの文字）を実装。
    - **文字の視認性・コントラスト重視の設計**:
        - ライトモード時の「受電」ノード（アクティブ状態）は、ブランチボタンと同様にパステル調の青背景（`#e6f7ff`）に濃い青文字（`#1890ff`）、淡い青境界（`1px solid #91d5ff`）を適用し、青みの強さを抑えつつ文字を際立たせる設計。
        - **協力会社ノード内の個別テキスト文字色**:
            - ライトモード時に「協力会社」カテゴリの下にある「AFC、BLP等」および「FRESH ROOM、設置業者等」のテキストが白背景と同化するのを防ぐため、独自カラーや白のハードコードを廃止し、テーマ連動の文字色変数 `var(--text)` を適用。ライトモード時は黒系、ダークモード時は白系に自動で切り替える。
        - **履歴カード（個人メモ履歴・不通履歴）の視認性向上**:
            - ライトモード時、白半透明のサイドバー背景と同化しやすい履歴カードの背景色（`--history-item-bg` / `--personal-memo-item-bg`）を、コントラストのある薄い青グレー（`rgba(24, 144, 255, 0.05)`〜`rgba(24, 144, 255, 0.06)`）に変更。
            - 履歴カード（`.history-item`）は、非ホバー時でも境界線 `border: 1px solid var(--glass-border)`（ライト時は `rgba(0, 0, 0, 0.12)`）を最初から表示する設計とし、輪郭を際立たせることで完全な同化を防ぐ。
        - **サイドバー内個人メモの「履歴」トグルボタンの視認性改善**:
            - メモ入力欄下部にある「個人メモとして保存」ボタンの右隣にある「履歴」トグルボタン（`.memo-history-toggle`）について、背景・文字・ホバー時のカラーが白基調にハードコードされていたためライトモードで文字が完全に同化していた問題を解消。背景を `var(--sub-link-bg)`、文字色を `var(--text)`、ホバー時の背景を `var(--sub-link-hover-bg)` へテーマ連動化し、両テーマで調和し明瞭に可読・クリックできるデザインへ変更。
        - **入力欄（メモ入力欄・架電メモ入力欄）の背景色の改善**:
            - ライトモード時、入力欄の背景色（`--input-bg`）が薄いグレー（`rgba(0, 0, 0, 0.06)`）であったため「入力不可（disabled）」と見間違える可能性があった問題を修正。ライトモード時の入力背景色を白（`#ffffff`）に設計変更し、アクティブに入力可能なテキストボックスであることが視覚的に一目で伝わるように最適化。また、React版のベース `.memo-input` クラスが持つハードコードされた背景色や文字色を、ダーク/ライト両テーマに対応したCSS変数（`var(--input-bg)`、`var(--input-text)`）に置き換え、テーマ切り替え時の整合性を担保。
        - ダークモードおよびライトモードにおける各テキスト（説明文、ヒント文、プレースホルダー）や各種入力欄の同化を完全に防ぐため、コントラスト比を高めた配色設計。
        - 下部バーの「架電者」バッジは、背景色とボーダーの不透明度を調整し、文字サイズを少し拡大・強調。ライトモードでは濃い青字（`#1890ff`/`#0050b3`）、ダークモードでははっきりした白字（`#ffffff`）にすることで、一目で識別可能な高視認性を確保。
        - 下部バーの「履歴」ボタン（不通の右隣）の配色を「集計ダッシュボード」ボタンと同様のテーマ用配色（`var(--dashboard-trigger-*)`）に統一し、ライトモードで文字が白背景と同化して見えなくなる問題を完全に解消。
        - 下部バーから展開する「不通履歴パネル」および「個人メモ履歴一覧パネル」の配色を「集計ダッシュボード」と同様のテーマ用配色（`var(--dashboard-bg)`、上部の太めのプライマリアクセント枠線、テーマ別の影 `var(--card-shadow)` 等）に統一し、ライトモード/ダークモードそれぞれの画面で文字が完全に同化しない美しい統一デザインへ改善。
        - 用件詳細サイドバーの「対応完了として記録」ボタンについて、主張の強すぎた「不透明な黄緑ベタ塗りと派手な影」を廃止。テーマ別に調和する半透明の緑背景（`var(--success-bg)`）と細い枠線（`var(--success-border)`）を使用したアウトライン基調のグラスモーフィズムデザインへ変更し、全体のレイアウトへシームレスに馴染むよう改良。
- **転送画面の廃止と引き継ぎ記録画面への移行**:
    - 転送の仕組みがないため、従来の用件選択完了後の「○○へ転送中...」という最終画面を廃止。代わりに「○○への引き継ぎを記録しました」と表示し、対応完了を記録する画面へ変更。

## 受電統計ダッシュボード機能

オペレーターの受電対応データを蓄積し、ビジュアルに集計・可視化する機能。画面下部メモバーの「個人メモ履歴」ボタンの左隣（スタンドアロンHTML版では架電メモ「履歴」ボタンの右隣）の「集計ダッシュボード」ボタンから起動可能。

1. **自動ログ記録**:
    - 詳細ヒアリング画面での「対応完了を記録」ボタン押下、または最終完了画面（「引き継ぎを記録しました」画面）への遷移時に、以下のデータを自動的に記録。
        - **受電者 (オペレーター名)**: ログインユーザー名（Microsoft Entra ID）。HTML版（非ログイン時）は架電メモの「架電者」入力値、または「ローカルユーザー」。
        - **発信元 (誰から)**: カテゴリ・ノードIDから自動判別（例: 管理会社、管理員、協力会社、建築、連動相手、居住者）。
        - **宛先 (どこ宛)**: 引き継ぎ先カテゴリから自動判別（例: ディスパッチャー、保守管理、施工管理、他部署）。
        - **用件**: 選択された具体的な用件テキスト。
        - **日時**: 記録された日付時刻。
2. **ログ保存先**:
    - React版: Firebase Cloud Firestore (`call_logs` コレクション)
    - スタンドアロンHTML版: `localStorage` (`talk_script_call_logs` キー)
3. **ダッシュボード表示内容**:
    - **総受電件数**: 指定された期間内の受電メモ保存総数。
    - **発信元の内訳**: 「誰から（発信元カテゴリ別）」の受電状況と割合（カラーバーグラフ表示）。
    - **引き継ぎ先の内訳**: 「どこ宛（宛先カテゴリ別）」の受電状況と割合（カラーバーグラフ表示）。
    - **オペレーター別の対応件数**: 各オペレーターの対応件数をフラットなリスト（優劣を排除したシンプルなドットマーク）で表示。
    - **用件別の受電件数**: 各用件の発生件数をフラットなリスト（優劣を排除したシンプルなドットマーク）で表示。
4. **期間フィルタ**:
    - 「本日」「直近7日間」「直近30日間」「全期間」の4パターンで動的に集計データを切り替え可能。
5. **統計データのリセット（管理者限定）**:
    - 特定の管理者権限を持つユーザーのみ、これまでの全統計データを一括で消去し、リセットする機能。
    - **判定基準**:
        - React版: ログインメールアドレスが環境変数 `VITE_ADMIN_EMAILS` に設定されているか、またはアカウント情報に `tomimoto`（友本）や `000644` などの管理者情報が含まれる場合。
        - スタンドアロンHTML版: 架電メモの「架電者」入力欄に `tomimoto`（友本）、`管理者`、`admin` または `000644` が入力されている場合。
